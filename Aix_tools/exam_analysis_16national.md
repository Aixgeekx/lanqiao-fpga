# 第16届国赛题目解析

## 题目来源
第16届蓝桥杯FPGA赛道国赛题目

---

## 一、题目概述

基于FPGA设计实现**动态数据采集、处理与交互系统**：
- ADC产生128个数字量（0-127）
- 按键触发数据录入与存储管理
- 集成4种算法：极值统计、无损压缩、滑动平均滤波、数据归一化
- 数码管显示结果，串口上报数据

## 二、系统架构

### 2.1 五层模块划分

| 层级 | 功能 | 关键模块 |
|------|------|---------|
| 输入采样层 | ADC读取、按键消抖 | adc_ctrl, key_ctrl |
| 时序控制层 | 界面状态机、算法状态机 | display_fsm, calc_fsm |
| 数据存储层 | 数据数组(≥16)、结果存储 | data_mem, result_mem |
| 算法层 | 极值/压缩/滤波/归一化 | algorithm_unit |
| 显示输出层 | 数码管、LED、UART | seg_proc, uart_sender |

### 2.2 双状态机设计

**界面状态机**：
```
INPUT → RESULT（通过S2切换算法，S4触发计算后跳RESULT）
RESULT → INPUT（S4清除数据后跳INPUT）
```

**算法状态机**：
```
IDLE → CALC → DONE（S4触发，多拍扫描完成后置done）
```

## 三、功能要求详解

### 3.1 数据录入

- ADC081C021：I2C地址1010100，8位输出（0-127）
- 电位器0~3.3V → 数字量0~127
- S1按下时把当前ADC值写入data_mem[wr_ptr]
- 同时记录RTC时间戳（DS1302读取）

### 3.2 四种算法

#### ① 极值统计
```
输入：[62,88,88,99,99,9,8,88]
输出：MIN=(8,6), MAX=(99,3)
```
实现：遍历数组，两个寄存器记录当前最小/最大值及其位置。O(n)时间。

#### ② 无损压缩（游程编码）
```
输入：[88,88,88,99,99,9,8,88]
输出：(88,3),(99,2),9,8,88
```
实现：顺序扫描，当前值与前值相同则计数+1，不同则输出(值,计数)对。游程≥2输出对，否则直接输出值。

#### ③ 滑动平均滤波
```
窗口=3，保留整数
输入：[62,88,88,99,99,9,8,88]
输出：[79,91,95,69,38,35]
```
实现：对每个位置i(1~N-2)，取(data[i-1]+data[i]+data[i+1])/3取整。输出长度=N-2。

#### ④ 最大-最小归一化
```
归一化到[0,1]，保留2位小数
输入：[127,8,10,2,8]
输出：[1.00,0.05,0.06,0.00,0.05]
```
实现：(value-min)/(max-min)。定点数：先×100再÷(max-min)。注意max==min时除零保护。

### 3.3 显示功能

**录入界面**：
- 界面标识（如"IN"）
- 最近录入的数字量
- 当前ADC转换值

**结果界面**：
- 算法编号（A1/A2/A3/A4）
- 算法结果数据

S3按键在结果界面切换显示各条结果。

### 3.4 按键功能

| 按键 | 录入界面 | 结果界面 |
|------|---------|---------|
| S1 | 录入当前ADC值 | 无定义 |
| S2 | 切换算法编号 | 切换算法编号 |
| S3 | 串口上报已录入数据 | 切换显示结果 |
| S4 | 执行计算 | 清除数据返回录入 |
| RESET | 全局复位 | 全局复位 |

### 3.5 串口上报格式

115200/8N1，字符串格式输出。

**录入数据上报**（S3触发）：
```
[时间戳][数值][时间戳][数值]...
示例：[135230][127][135235][8][135240][10]
```

**算法结果上报**（计算完成触发）：
```
极值：[时间戳][A1][(最小值,位置),(最大值,位置)]
压缩：[时间戳][A2][(数值,游程长度),数值...]
滤波：[时间戳][A3][滤波结果序列]
归一化：[时间戳][A4][0.xx结果序列]
```

## 四、实现顺序建议

1. **ADC读取**：I2C读ADC081C021，数码管显示当前值
2. **按键消抖**：S1-S4消抖，LED显示按下状态
3. **数据录入**：S1把ADC值写入数组，数码管显示已录入数量
4. **界面切换**：S2切换算法编号，数码管显示A1/A2/A3/A4
5. **极值统计**：先实现最简单的算法，S4触发，结果存入寄存器
6. **结果界面**：数码管显示极值结果，S3切换MIN/MAX显示
7. **串口上报**：S3在录入界面触发数据上报
8. **其他算法**：逐个实现压缩、滤波、归一化
9. **RTC时间戳**：DS1302读取时间，格式化到串口输出
10. **界面切换完善**：录入/结果界面的完整切换逻辑

## 五、常见易错点

1. **数组深度**：至少16个，建议按128设计（题目说128个数字量）
2. **算法多拍**：不要一个时钟写完所有循环，用状态机逐元素处理
3. **除零保护**：归一化时max==min的情况要特殊处理
4. **定点数精度**：×100后再除，用整数运算避免浮点
5. **串口格式化**：先把结果转成ASCII字符串，再逐字节发送
6. **wr_ptr管理**：录入时递增，清零时归零
7. **done标志**：算法完成后置done，界面状态机根据done跳转
8. **S2任意界面有效**：不同于S1/S3/S4的界面限制
